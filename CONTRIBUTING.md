# Contributing

Atelier OS wird derzeit als privates Langzeitprojekt entwickelt.

## Grundregel

Keine neue Funktion wird spontan in eine laufende Version aufgenommen.

## Änderungsablauf

1. Problem oder Idee als Issue erfassen.
2. Ziel und Nicht-Ziel benennen.
3. Auswirkungen auf Datenmodell und bestehende Objekte prüfen.
4. Entscheidung dokumentieren.
5. Änderung umsetzen.
6. Changelog aktualisieren.
7. Mit klarer Commit-Nachricht speichern.

## Commit-Nachrichten

Beispiele:

```text
docs: define context-before-editing principle
vault: add scene template
example: connect scene 4 to canon and decisions
fix: correct invalid object relation
```

## Definition of Done

Eine Änderung ist erst fertig, wenn:

- sie dokumentiert ist,
- bestehende Daten nicht unbegründet bricht,
- ein nachvollziehbarer Test oder Prüfschritt existiert,
- und das Changelog bei relevanten Änderungen aktualisiert wurde.
