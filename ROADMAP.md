# Roadmap

Die Roadmap trennt bewusst aktuelle Arbeit von späteren Ideen.

## Sprint 0 — Repository Foundation

**Ziel:** Das Projekt ist verständlich, dokumentiert und versionierbar.

- [x] Repository-Struktur
- [x] README
- [x] Manifest
- [x] Architekturgrundlagen
- [x] Objektmodell
- [x] Beziehungen
- [x] erste ADRs
- [x] Vault-Spezifikation

## Sprint 1 — Nutzbare Vault

**Ziel:** Atelier OS kann täglich in Obsidian genutzt werden.

- [ ] bestehende Vault konsolidieren
- [ ] zentrales Dashboard
- [ ] Dataview-Abfragen
- [ ] Objekt-Templates
- [ ] Kontextblock pro zentralem Objekt
- [ ] Experiment Null als vollständiges Beispiel
- [ ] Szene 4 als Referenz-Workspace
- [ ] Installations- und Testanleitung

## Sprint 2 — Creative Library

**Ziel:** Musik, Sounds, Instrumente und Wissen werden gleichwertig unterstützt.

- [ ] Album-, Song- und Demo-Objekte
- [ ] Soundbibliothek
- [ ] Synthesizer- und Effektgeräte-Inventar
- [ ] Kunsttheorie und Wissensbereiche
- [ ] Audio- und Bildreferenzen

## Sprint 3 — Memory und Qualitätssicherung

**Ziel:** Änderungen und Kontextverluste werden systematisch verhindert.

- [ ] Objektchroniken
- [ ] Entscheidungsregister
- [ ] Kontextprüfungen
- [ ] Validierung von IDs und Beziehungen
- [ ] automatisierte Repository-Checks

## Späteres Backlog

- Companion-Oberfläche
- Assistenten-Workflows
- semantische Suche
- eigene Desktop-App
- API
- Graph Explorer

Diese Punkte werden nicht in laufende Sprints gezogen, ohne zuvor die Spezifikation zu ändern.
