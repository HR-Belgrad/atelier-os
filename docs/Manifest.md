# Manifest

## 1. Das Projekt ist das Gedächtnis

Atelier OS darf nicht vom Erinnerungsvermögen eines Chats, einer Person oder eines einzelnen Programms abhängen. Relevante Entscheidungen, Begründungen und Beziehungen müssen im Projekt selbst auffindbar sein.

## 2. Wissen ist ein Netz

Ordner helfen beim Navigieren, bilden aber nicht die eigentliche Bedeutung ab. Bedeutung entsteht durch benannte Beziehungen zwischen Objekten.

## 3. Es gibt keine bedeutungslosen Notizen

Eine Datei, die dauerhaft Teil des Systems wird, soll als Objekt oder als klar zugeordnetes Dokument verstanden werden.

## 4. Information wird möglichst nur einmal gepflegt

Mehrfachkopien derselben Aussage führen zu Widersprüchen. Stattdessen wird auf die maßgebliche Quelle verwiesen.

## 5. Entscheidungen gehören zum Werk

Nicht nur Ergebnisse, sondern auch Varianten, verworfene Wege und Begründungen sind wertvoll. Sie werden erhalten, ohne den aktuellen Kanon zu verwischen.

## 6. Kontext wird vor der Bearbeitung geladen

Jedes zentrale Objekt kann eine Pflichtlektüre definieren. Wer an einer Szene, einem Song oder einem Werk arbeitet, soll zuerst den relevanten Kanon, die Entscheidungen, die Historie und offene Fragen sehen.

## 7. Speichern ist eine bewusste Handlung

Eine Idee aus einem Gespräch wird nicht automatisch zum dauerhaften Bestandteil des Systems. Die ausdrückliche Freigabe **„Speichern“** markiert den Commit in das kreative Gedächtnis.

## 8. Geschichte wird nicht stillschweigend gelöscht

Änderungen werden versioniert. Frühere Stände bleiben über Git oder Objektchroniken nachvollziehbar.

## 9. Offene Formate haben Vorrang

Markdown, YAML und gewöhnliche Medienformate bilden die Grundlage. Die Oberfläche darf austauschbar bleiben.

## 10. Die Technik dient der Arbeit

Atelier OS ist kein Selbstzweck. Neue Funktionen müssen einen konkreten kreativen Arbeitsablauf verbessern.
