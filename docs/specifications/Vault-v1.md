# Spezifikation: erste nutzbare Vault

## Status

Draft für Sprint 1.

## Ziel

Die bestehende Atelier-OS-Vault wird zu einem täglich nutzbaren Kreativsystem, das relevanten Kontext sichtbar macht und zentrale Übersichten automatisch erzeugt.

## Lieferumfang

### 1. Zentrales Dashboard

Das Dashboard zeigt ausschließlich:

- aktive Projekte
- offene Aufgaben
- zuletzt geänderte Objekte
- neue Ideen
- Inbox

### 2. Objektkarten

Templates und Beispielkarten für:

- Werk
- Szene
- Song
- Sound
- Synthesizer
- Wissensartikel
- Motiv
- Figur
- Entscheidung

### 3. Dataview

Automatische Ansichten für:

- Werke
- Songs
- Sounds
- Figuren
- Motive
- Synthesizer
- Entscheidungen
- Speicherpunkte

### 4. Kontextblock

Jedes zentrale Arbeitsobjekt besitzt einen sichtbaren Abschnitt **„Vor Bearbeitung lesen“**.

### 5. Referenzprojekt Experiment Null

Experiment Null wird als vollständiges Beispiel eingebunden. Szene 4 demonstriert:

- Werkbezug
- Kanon
- Entscheidungen
- Varianten
- Denkprozess
- Quellen
- Audio
- offene Fragen
- Pflichtkontext

### 6. Dokumentation

- Installation
- Dataview-Einrichtung
- Template-Nutzung
- täglicher Workflow
- Testanleitung

## Nicht enthalten

- eigene Desktop-App
- KI-Agenten
- Companion
- API
- Graph-Datenbank
- Cloud-Backend
- automatische KI-Änderungen

## Abnahmekriterien

- Vault öffnet ohne strukturelle Fehler.
- Dataview-Abfragen liefern anhand der Beispieldaten Ergebnisse.
- Alle vorgesehenen Templates sind vorhanden.
- Szene 4 verlinkt den vollständigen Pflichtkontext.
- Ein neuer Objekttyp kann anhand der Dokumentation korrekt angelegt werden.
- Ein manueller Testplan deckt die Kernabläufe ab.
