# Benennung

## Dateinamen

Dateinamen bleiben menschenlesbar. Umlaute und Leerzeichen sind grundsätzlich erlaubt, solange Werkzeuge damit zuverlässig umgehen.

Beispiele:

```text
Szene 4.md
Roland SH-101.md
Papier.md
```

## IDs

IDs sind technisch stabil und folgen diesem Muster:

```text
<type>-<name>-<optional-context>
```

Beispiele:

```text
scene-experiment-null-004
song-night-train
sound-sh101-hollow-bass
```

## Ordner

Ordner dienen der Orientierung, nicht der Identität. Eine Verschiebung in einen anderen Ordner darf die Objekt-ID nicht ändern.

## Sprache

Die sichtbaren Inhalte dürfen deutsch sein. Technische Feldnamen und Beziehungsnamen werden zunächst englisch gehalten, damit spätere Werkzeuge einfacher anschließen können.
